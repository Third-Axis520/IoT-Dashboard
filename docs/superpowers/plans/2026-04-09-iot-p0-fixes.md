## Task 2: 修復 SseHub 廣播 — 並行發送 + Timeout

**Files:**
- Modify: `backend/Services/SseHub.cs:30-79`

**問題：** `BroadcastAsync` 和 `SendHeartbeatAsync` 對所有連線**逐個** await，若某個連線的 `response.WriteAsync` 卡住（客戶端緩衝滿），整個廣播迴圈會停在那裡，所有其他連線都要等。

**解法：** 改為 `Task.WhenAll`，每個連線用獨立 Task，並加 5 秒 timeout。

- [ ] **Step 1: 新增 helper 方法 `WriteToConnectionAsync`**

在 `SseHub.cs` 的 class 內新增私有方法：

```csharp
/// <summary>向單一連線寫入訊息，5 秒 timeout，失敗時回傳 connectionId 供移除。</summary>
private static async Task<string?> WriteToConnectionAsync(
    string id, HttpResponse response, string message, CancellationToken ct)
{
    try
    {
        using var cts = CancellationTokenSource.CreateLinkedTokenSource(ct);
        cts.CancelAfter(TimeSpan.FromSeconds(5));

        await response.WriteAsync(message, cts.Token);
        await response.Body.FlushAsync(cts.Token);
        return null; // 成功，不需移除
    }
    catch
    {
        return id; // 失敗，回傳 id 供移除
    }
}
```

- [ ] **Step 2: 改寫 `BroadcastAsync`**

```csharp
public async Task BroadcastAsync(object payload, CancellationToken ct = default)
{
    if (_connections.IsEmpty) return;

    var json = JsonSerializer.Serialize(payload, _jsonOptions);
    var message = $"event: data-update\ndata: {json}\n\n";

    var tasks = _connections.Select(kv =>
        WriteToConnectionAsync(kv.Key, kv.Value, message, ct));

    var results = await Task.WhenAll(tasks);

    foreach (var deadId in results.Where(id => id != null))
        _connections.TryRemove(deadId!, out _);
}
```

- [ ] **Step 3: 改寫 `SendHeartbeatAsync`**

```csharp
public async Task SendHeartbeatAsync(CancellationToken ct = default)
{
    if (_connections.IsEmpty) return;

    const string message = "event: heartbeat\ndata: {}\n\n";

    var tasks = _connections.Select(kv =>
        WriteToConnectionAsync(kv.Key, kv.Value, message, ct));

    var results = await Task.WhenAll(tasks);

    foreach (var deadId in results.Where(id => id != null))
        _connections.TryRemove(deadId!, out _);
}
```

## Task 3: 前端 localStorage 多頁簽同步

**Files:**
- Modify: `frontend/src/App.tsx:119-131`

**問題：** 同一使用者在兩個分頁開啟 Dashboard，兩邊都監聽自己的 SSE、各自修改 state、2 秒 debounce 後各自存 localStorage。結果：最後存的一方覆蓋另一方，設備配置可能丟失。

**解法：** 使用瀏覽器原生 `BroadcastChannel` API。當某個頁簽寫入 localStorage 後，廣播通知其他頁簽同步，其他頁簽收到後重新從 localStorage 讀取（不覆蓋，只更新為最新版本）。

**不需引入新依賴**，BroadcastChannel 是現代瀏覽器原生 API（Chrome 54+, Firefox 38+, Safari 15.4+）。

- [ ] **Step 1: 在 App.tsx 加 BroadcastChannel ref**

在 `App.tsx` state 宣告區段（約第 92 行後），加入：

```tsx
const bcRef = useRef<BroadcastChannel | null>(null);

useEffect(() => {
  bcRef.current = new BroadcastChannel('iot-dashboard-sync');
  bcRef.current.onmessage = (evt) => {
    if (evt.data?.type === 'DATA_UPDATED') {
      try {
        const stored = localStorage.getItem(DATA_STORAGE_KEY);
        if (stored) setData(migrateStoredData(JSON.parse(stored)));
      } catch { /* ignore */ }
    }
  };
  return () => bcRef.current?.close();
}, []); // eslint-disable-line react-hooks/exhaustive-deps
```

- [ ] **Step 2: 在 data 持久化的 useEffect 裡，存完後廣播通知**

找到現有的 data 持久化 useEffect（約第 123-131 行）：

```tsx
// 改前
persistTimerRef.current = setTimeout(() => {
  try { localStorage.setItem(DATA_STORAGE_KEY, JSON.stringify(serializeForStorage(data))); } catch { /* quota */ }
```
