# IoT Dashboard 系統操作指南

## 系統架構概覽

系統採用**全 UI 操作、DB 儲存**的彈性設計。所有產線、設備、感測器設定皆透過介面完成，不需修改程式碼。

| 資料類型 | 儲存位置 | 修改方式 |
|---------|---------|---------|
| 產線 / 設備配置 | 後端 DB | UI 操作 |
| 感測器屬性類型 | 後端 DB | UI → 屬性類型 |
| 設備連線（PLC/API） | 後端 DB | 設備整合精靈 |
| UCL / LCL 限值 | 後端 DB | UI → 限值設定 |
| 鞋子在位偵測 | 後端 DB | 綁定 `material_detect` 屬性 |

> **注意：** 目前前端仍保留一個舊的「烤箱生產線」Tab（硬編碼在 `liveLineConfig.ts`）。
> 正確做法是透過本指南的流程，用 UI 重新建立它，之後再從程式碼移除舊 Tab。

## 第一部分：建立新產線

### 1-1 新增產線

1. 點左上角**產線下拉選單**
2. 滾到底部 → **＋ 添加产线**
3. 輸入產線名稱（例如「烤箱生產線」）→ 按 Enter

### 1-2 刪除產線

1. 開啟**編輯模式**（右上角鎖頭圖示）
2. 點產線下拉 → 滑鼠移到產線名稱 → 出現垃圾桶圖示 → 點擊刪除

---

## 第二部分：新增設備到產線（設備整合精靈）

每台真實 PLC 設備都透過**設備整合精靈**建立，共 7 步。

### 開啟精靈

右上角下拉 → **整合新設備**

### Step 1：選擇通訊協議

| 協議 | 適用情境 |
|------|---------|
| **Modbus TCP** | 標準 PLC（西門子、三菱、台達等） |
| **Web API** | 設備提供 REST API |
| **Push Ingest** | 設備主動推送（MQTT gateway 等） |

### Step 2：填入連線設定

**Modbus TCP：**
```
主機位址：設備 IP（例如 192.168.1.100）
Port：502
單元 ID：1（大多數設備預設）
```

**Web API：**
```
端點 URL：http://設備IP/api/data
HTTP 方法：GET
JSON 路徑：data.value（點號路徑）
輪詢間隔：5000（毫秒）
```

### Step 3：掃描設備資料點

點 **掃描設備** → 系統列出所有可用暫存器 → 勾選要監控的點。

### Step 4：選取資料點

從掃描結果中勾選要監控的暫存器。

### Step 5：貼屬性標籤

每個選取的資料點設定：
- **屬性類型**：溫度 / 壓力 / 濕度 / 流量 / 計數器 / 狀態 / 在位
- **顯示名稱**：自訂（例如「大底溫度」）
- **單位**：℃ / kPa / % 等

### Step 6：設定設備類型

```
設備類型名稱：例如「藥水箱」
顯示類型：
  single_kpi      — 單一大數字（1 個感測點）
  dual_side_spark — 左右兩組（2-3 個感測點）
  four_rings      — 四個環形指針（4 個感測點）
  molding_matrix  — 矩陣格式（6 個感測點）
```

### Step 7：確認建立

點 **建立** → 系統自動：
1. 建立 DeviceConnection（開始 Polling）
2. 建立 EquipmentType（設備類型範本）
3. SSE 推送更新到所有瀏覽器

### 精靈完成後：加入產線

精靈建立的是「設備類型範本」，還需手動加入產線：

1. 右上角下拉 → **加入既有類型**
2. 選取剛建立的設備類型
3. 完成 → 設備卡片出現在儀表板

## 第三部分：手動建立「烤箱生產線」— 實際資料

以下是把原本硬編碼的烤箱生產線，改用 UI 重新建立的完整資料與流程。

### PLC 連線設定（來自 OvenDataReceive 系統）

| 欄位 | 值 | 精靈填入位置 |
|------|---|------------|
| 主機位址 | `192.168.61.74` | Step 2 主機位址 |
| Port | `502` | Step 2 Port |
| 溫度感測器 Unit ID | `1` | Step 2 單元 ID |
| DI 感測器 Unit ID | `1` | Step 2 DI Unit ID |
| 連線名稱 | `烤箱生產線` | Step 2 連線名稱 |
| 資產編號 (AssetCode) | `OVEN-LINE-01` | 加入產線時設備綁定 |

> **RS485 Daisy Chain 注意：** 感測器 2、3（藥水箱）是 RS485 串聯鏈的起點。若藥水箱斷電或斷線，感測器 4-12 都會無法讀取。

### 溫度感測器 Register 對照表（40001 起始，連續排列）

所有溫度感測器從 register `40001` 開始，每個感測器佔 1 個 register，依序排列：

| Register | SensorId | 設備 | 測量點 | UCL | LCL | 單位 |
|----------|----------|------|--------|-----|-----|------|
| **40001** | 1 | 高速加熱定型機 | 設備溫度 | 185 | 130 | ℃ |
| **40002** | 2 | 藥水箱 | 大底溫度 | 85 | 55 | ℃ |
| **40003** | 3 | 藥水箱 | 鞋面溫度 | 80 | 50 | ℃ |
| **40004** | 4 | 一次膠 | 大底溫度 | 80 | 50 | ℃ |
| **40005** | 5 | 一次膠 | 鞋面溫度 | 75 | 45 | ℃ |
| **40006** | 6 | 二次膠 | 大底溫度 | 80 | 50 | ℃ |
| **40007** | 7 | 二次膠 | 鞋面溫度 | 75 | 45 | ℃ |
| **40008** | 8 | 冷凍機 | 設備溫度 | -10 | -30 | ℃ |
| **40009** | 9 | 後跟定型 | 熱定型右 | 135 | 100 | ℃ |
| **40010** | 10 | 後跟定型 | 冷定型右 | 20 | 3 | ℃ |
| **40011** | 11 | 後跟定型 | 熱定型左 | 135 | 100 | ℃ |
| **40012** | 12 | 後跟定型 | 冷定型左 | 20 | 3 | ℃ |

> **資料格式：** 原始值為 16-bit unsigned，先做 byte swap，再乘以 0.1 得實際溫度。  
> 例：原始值 256 → byte swap → 0x0100 → 256 → ×0.1 → **25.6°C**

### 鞋子在位偵測（DI 數位輸入）

原系統使用 **32 個 DI 數位輸入**（Function Code 02，從 address 0 開始讀 32 個 bit）偵測在位狀態，並非 register 40013。

在精靈中設定方式：
- 協議選 Modbus TCP，掃描後選取對應的 DI channel
- Step 5 屬性類型選「**在位**」
- 系統自動識別為鞋子在位偵測，無需手動指定 SensorId

---

### 操作流程

1. 左上角產線下拉 → ＋ → 輸入「烤箱生產線」→ Enter
2. 右上角下拉 → **整合新設備** → 跑精靈（共 6 次，每台一次）
   - Step 1：Modbus TCP
   - Step 2：主機 `192.168.61.74`、Port `502`、Unit ID `1`
   - Step 3：掃描後依上表選取對應 register（40001-40012）
   - Step 5：依上表設定屬性類型 + 顯示名稱
   - Step 6：填設備類型名稱 + 顯示類型（依上表）
   - Step 7：建立
3. 精靈完成後 → **加入既有類型** → 選剛建的設備 → 加入烤箱生產線
4. 加入時設備綁定 AssetCode：`OVEN-LINE-01`
5. 重複步驟 2-4，直到 6 台設備全部加入
6. 右上角 🎚 → **限值設定** → 依各設備表格填入 UCL / LCL → 儲存

## 第五部分：屬性類型管理

### 系統內建 8 個屬性類型（不可刪除）

| 名稱 | Behavior | 預設單位 | 說明 |
|------|---------|---------|------|
| 溫度 | normal | ℃ | 一般量測值 |
| 壓力 | normal | kPa | 一般量測值 |
| 濕度 | normal | % | 一般量測值 |
| 流量 | normal | L/min | 一般量測值 |
| 計數器 | counter | count | 累計計數 |
| 狀態 | state | — | 開關狀態 |
| 資產編號 | asset_code | — | 設備識別碼 |
| 在位 | material_detect | — | 鞋子在位偵測 |

> **在位偵測說明：** 只要某個感測點綁定了「在位」屬性類型，系統自動識別為在位感測器（原 PLC register 40013），不需硬編碼 SensorId。

### 新增自訂屬性類型

右上角 **⚙️ 系統設定** → **屬性管理** → **新增** → 填入名稱、圖示、預設單位、UCL/LCL 預設值

新增後在精靈 Step 5 的下拉清單中出現。

## 第六部分：系統監控

### 查看設備連線健康狀態

右上角 **⚙️ 系統設定** → **連線管理** → 顯示所有 DeviceConnection 狀態

或直接查 API：
```
GET /api/diagnostics/polling
```

回傳每個設備的：連線狀態（healthy / error / disabled）、連續錯誤次數、最後 Poll 時間、錯誤訊息

### Circuit Breaker 機制

- 連續 **3 次** Poll 失敗 → 切換慢速重試（每 30 秒一次）
- 恢復成功後自動回到正常間隔

### SSE 即時推送

設備連線、屬性類型等設定變更 → 自動推送到所有已開啟的瀏覽器，無需手動重新整理。
