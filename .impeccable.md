# Design Context — IoT Dashboard

> 由 teach-impeccable 建立。所有設計 Skill 以此為基準。

---

## Design Context

### Users

三個角色共用同一系統，場景各異：

- **製造現場操作員**：站在機台旁，眼睛看壁掛大螢幕，手邊可能有工具，不會細讀文字，靠顏色和形狀判斷狀態
- **製程工程師**：辦公室和現場兩邊跑，需要遠端查看趨勢、設定限值、排查異常，會操作較多功能
- **廠長 / 管理層**：偶爾查看 KPI 和告警，不操作設備，只需要一眼知道「有沒有問題」

**主要顯示環境**：工廠壁掛大螢幕（固定裝置，遠距觀看）+ 辦公室桌機 1080p 以上。**不是**行動裝置優先。

**核心工作**：即時溫控監控（超限立即感知）、趨勢確認、告警確認、設備設定維護。

---

### Brand Personality

三個詞：**精準、即時、掌控感**

語氣：冷靜而有力，像工業儀器而非消費品。沒有「歡迎使用」的溫暖感，有的是「一切盡在掌握」的穩定感。

情感目標：操作員看到這個介面應該感到**放心和有把握**，而非興奮或娛樂。異常發生時應感到**緊迫但不慌亂**（因為系統清楚告訴他發生了什麼）。

---

### Aesthetic Direction

**定位**：工業科幻指揮台（Industrial Sci-Fi Command Center）

介於 SCADA 工業系統的嚴謹與科幻電影控制室的視覺震撼之間。不是消費性 App，不是 ERP，是為精密製造設計的高密度監控介面。

**視覺語言（已建立，須延續）**：
- 深海軍藍底色（`#050B14`）+ 電光青主題色（`#00E5FF`）
- 螢光綠 = 正常（`#00FF66`）、危險紅 = 告警（`#FF1744`）、黃 = 警告（`#FFEA00`）
- 玻璃擬物面板（自製 glass-panel / glass-card CSS classes）
- 告警狀態：旋轉發光邊框 + 脈衝動畫（功能性，非裝飾性）
- 數值一律 Roboto Mono，中文標籤用 Inter
- Dark 模式為主，Light 模式完整支援

**反面參考（絕對禁止）**：
- 消費性 App 的圓潤粉彩卡片風（Instagram、Notion 感）
- 普通 ERP 的灰色無聊表格（SAP 感）
- 視覺層次過於扁平稀疏（過度 minimal）
- 用漸層文字當「科技感」裝飾（`gradient-text` 模板）
- 純裝飾性的動畫效果（動畫必須有功能意義）

---

### Design Principles

1. **零歧義優先**
   任何異常狀態必須在 3 公尺外的大螢幕也能一眼識別。告警顏色、脈衝動畫是功能，不是裝飾。「看起來好看」永遠輸給「看清楚了」。

2. **密度但不擁擠**
   操作員需要同時監控大量數值，但絕不能讓人感到找不到東西。資訊密度高，視覺層次更高。用間距和分組取代卡片嵌套。

3. **精密工程感**
   霓虹線條、等寬字體、暗底深藍是這個產品刻意選擇的工業 sci-fi 語言，不是 AI 偷懶的預設值。每個視覺元素都要感覺是「為了精密工業而設計的」。

4. **監控是主場景，設定是次要場景**
   儀表板永遠是第一優先。設定頁（暫存器對應、限值設定）求快、求清晰，不需要炫耀視覺，但必須與整體設計系統一致。

5. **遠近皆可讀**
   主要指標（設備狀態、溫度值、告警數）必須在壁掛大螢幕遠距清晰辨識。次要資訊（設備名稱、Sensor ID、小字說明）靠近才看得到是可接受的。

## Tech Constraints

- React 19 + TypeScript + Tailwind CSS 4 + Vite 6
- Charts: Recharts（不引入 D3 直接操作）
- Icons: Lucide React
- Animation: 純 CSS（無 Framer Motion，保持輕量）
- Fonts: Inter + Roboto Mono（Google Fonts CDN）
- No i18n library — UI 語言為繁中/簡中混用，設計需容納 CJK 字元
